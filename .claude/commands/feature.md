Create a feature branch for: $ARGUMENTS

Steps:
1. Check current git status - warn if uncommitted changes
2. Ensure we're on main and it's up to date
3. Create branch with name: feature/<kebab-case-description>
4. Switch to the new branch
5. Confirm the branch is ready for development

Branch naming:
- feature/ for new features
- fix/ for bug fixes
- chore/ for maintenance
- docs/ for documentation
